# p0-w4-d1-t3 · Пианино

> Тип: объёмная (генерация UI из данных + события) · Репозиторий: один отдельный

> Подключить `.js` к HTML через `<script>`, отсмотреть в браузере.

## Задача

Собрать пианино: клавиши генерируются из массива нот, клик подсвечивает клавишу и записывает ноту в последовательность, а потом записанное можно проиграть — подсветкой по очереди. Звук — опциональный релиз, основное здесь про генерацию клавиш и работу с массивом нажатий.

```
клавиши пианино из массива нот → DOM
┌──────────────────────────────────────────────────────┐
│ map: ноты → клавиши (белые/чёрные)                   │
│ клик по клавише → подсветка + запись в массив        │
│ воспроизвести записанное — подсветкой по очереди     │
└──────────────────────────────────────────────────────┘
звук — опционально, через готовые <audio>
```

## Дано

```html
<div id="piano"></div>
<div class="controls">
  <button id="play">Проиграть</button>
  <button id="clear">Очистить</button>
</div>
<p id="recorded">—</p>
```

```js
// одна октава: true = чёрная клавиша (диез)
const notes = [
  { note: "C", sharp: false }, { note: "C#", sharp: true },
  { note: "D", sharp: false }, { note: "D#", sharp: true },
  { note: "E", sharp: false }, { note: "F", sharp: false },
  { note: "F#", sharp: true }, { note: "G", sharp: false },
  { note: "G#", sharp: true }, { note: "A", sharp: false },
  { note: "A#", sharp: true }, { note: "B", sharp: false },
];
```

## Что нужно сделать (по релизам)

**Release 1 — клавиатура.** Через `map` сгенерировать клавиши из `notes`: белые и чёрные (класс по `sharp`), отрисовать в `#piano`. Клавиши создаются из массива, а не верстаются руками.

**Release 2 — запись нажатий.** По клику на клавишу: подсветить её (на короткое время) и добавить ноту в массив-последовательность. В `#recorded` показывать записанное (`join` в строку вроде `C – E – G`).

**Release 3 — воспроизведение и очистка.** Кнопка «Проиграть» проходит по записанному массиву и подсвечивает клавиши по очереди с задержкой между ними (нота за нотой). Для задержки между нотами понадобится `setTimeout` — его на лекции не было, разберись сам(а), как отложить действие по времени (ищи «setTimeout javascript»). «Очистить» сбрасывает последовательность. Ноту с клавишей связать через `data-`-атрибут.

## Опциональный релиз — звук

Если захочешь звук — подключить готовые короткие аудиофайлы нот через `<audio>` и проигрывать при нажатии. Web Audio и синтез звука не нужны и не проходили — только `<audio>` с готовыми сэмплами. Делать по желанию.

## Ожидаемый результат

Октава клавиш (белые + чёрные на своих местах). Клик подсвечивает клавишу и дописывает ноту: `#recorded` показывает `C – E – G`. «Проиграть» повторяет последовательность подсветкой. «Очистить» обнуляет.

## Критерии приёмки

* Клавиши сгенерированы `map`'ом из `notes`, чёрные/белые различаются классом; ручной верстки клавиш нет.
* Нажатия пишутся в массив, воспроизведение идёт по нему по очереди с задержкой; очистка работает.
* Нота связана с клавишей через `data-`-атрибут, обработка кликов — делегированием.
